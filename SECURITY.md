# Security Policy

## Supported Version

The latest `main` branch revision is supported.

## Reporting a Vulnerability

1. Use GitHub private security reporting when possible.
2. Otherwise open a `[security]` issue with sensitive exploit details removed.
3. Include tenant/auth context, impacted components, and reproduction path.

## Response Targets

- Initial acknowledgement: within 72 hours.
- Triage outcome and mitigation plan: within 7 days.

## Additional Notes

- See detailed control baseline in `docs/security.md`.
- Do not commit live tokens, secrets, or customer-like data.
